Significance of <head> section

19 july 2021

Introduction

When you first open your source-code editor such as Visual Studio Code you can easily get your hand on HTML boilerplate code such as the one shown below.

First, you will see The document type declaration (or doctype). A doctype is simply a way to tell the browser or any other parser what type of document it’s looking at. Second is the <html> element and from here on you will start writing your HTML code. The lang attributes with value en is specifying that the following document is in English. The <html> element is divided into two parts, the <head> and <body> sections. The <body> section contains everything that’s displayed in the browser — text, images, and so on. So, what is the significance of the <head> section...

<head> section

The <head> section is said to be a metadata holder. What is metadata? it is nothing but "data about the data(on the website)". This data is generally not shown to end-user but by using metadata correctly, you can boost your relevancy in search results. Metadata provides search engines with the most important information about your web pages, including titles and descriptions. When someone searches Google for an image, your metadata is what tells the search engine you have the photo in need.

Tags used in <head> section

<meta> tag: Used to specify a character set, page description, keywords, author of the document, and viewport settings. The first line inside the <head> section of an HTML document is the one that defines the character encoding for the document using this <meta> tag.

<title> tag: As its name suggests it specifies the title of the webpage which will appear on the browser tab which makes navigation in the browser easier. It provides the title for the web page when it is added to favorites.

<style> tag: You can put your style information(CSS) here. Although it is recommended to create a separate stylesheet.

<base> tag: The <base> HTML element specifies the base URL to use for all relative URLs in a document. You can just place the base URL in the head section and then all the subsequent relative links in the webpage regard it as starting point.There can be only one <base> element in a document.

<link> tag: This tag defines the relationship between the current document and external resources. The <link> element is an empty element, it contains attributes only. An easy example would of the CSS Stylesheet in this case we are using two attributes href & rel. It is also used for adding a favicon which is a small icon that appears in the browser tab.

<script> Tag:Used to implant or point toward the JavaScript code.

<noscript>Tag: This tag is used in those browsers only which does not support scripts. These tags can be used to let a user know that his browser cannot correctly display your pages that require JavaScript.
